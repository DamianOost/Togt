**Components:**

- **Schema (014):** `webhook_subscriptions` + `webhook_deliveries`, with `secret_previous_encrypted` + `secret_previous_expires_at` columns plus a paired CHECK constraint for the rotation grace window.
- **`lib/webhookSignature.js`:** Stripe-shape `t=<unix>,v1=<hmac_sha256_hex>`. Multi-v1 during the 24h rotation grace window — `signPayload(secret, body, ts?, previousSecret?)` emits both, `verifySignature` accepts any matching v1.
- **`lib/webhookSecretCrypto.js`:** AES-256-GCM with random 12-byte IV per write; `WEBHOOK_SECRET_ENCRYPTION_KEY` env (32 bytes hex), dev fallback `'a' * 64` with warning, prod fail-fast.
- **`lib/safeFetch.js`:** SSRF defence — `assertPublicHost(url)` blocks RFC1918, loopback, link-local (incl. `169.254.169.254` AWS IMDS), CGNAT, IPv6 ULA, IPv4-mapped IPv6. Gated by `NODE_ENV === 'production'` OR `WEBHOOK_SSRF_FORCE=1` so dev/test fixtures using 127.0.0.1 receivers continue to work. Applied at subscription-create AND dispatcher deliver-one (defence in depth — DNS could rebind between).
- **`services/events.js`:** `emitEvent(client, opts)` with REQUIRED client first arg (no default — footgun guard). 11 event types in `EVENT_TYPES`. Throws if anyone passes an unknown type or forgets the client.
- **`config/db.js`:** `withTx(fn)` helper added — `BEGIN`/run/`COMMIT` with `ROLLBACK`-on-throw and original-error preservation. The standard wrapper for any handler doing `UPDATE` + `emitEvent`.
- **`services/webhookDispatcher.js`:** Background loop polling at 5s. `FOR UPDATE SKIP LOCKED` claim with `attempt_count = attempt_count + 1 RETURNING *`. Retry curve [30s, 2m, 10m, 1h] then 1h-on-1h until age ≥ 24h → dead (~25 attempts/24h, mirrors Stripe). `axios.post` with `maxRedirects: 0` (SSRF amplifier). Per-tick `assertPublicHost` defence-in-depth. `last_response_body` no longer persisted — receiver-controlled storage was a PII / log-injection vector.
- **`routes/webhookSubscriptions.js`:** JWT-authed CRUD + `rotate-secret` + `deliveries` list + `replay`. Rotate refuses (409 `webhook-rotation-grace-active`) if a prior 24h grace window is still active. Rotate has a 5/10min/IP rate limit (`rotateSecretLimiter`) so a stolen JWT can't rotate-spam.
- **5 MCP tools:** `create_webhook_subscription` (`mcp:full`), `list_webhook_subscriptions` (`mcp:read_only`), `delete_webhook_subscription`, `rotate_webhook_secret`, `replay_webhook_delivery`. Each handler returns a `decision_context` block.
- **OpenAPI 3.1 + agents.json:** 5 paths, 4 schemas (`WebhookSubscription`, `WebhookSubscriptionWithSecret`, `WebhookEventEnvelope`, `WebhookDelivery`), 6 new `x-error-types`, `agents.json` declares the webhooks block with full retry policy + signature format + grace-window note.
- **Lifecycle wiring:** `emitEvent` calls at every transition — `services/matcher.js` (auto-match: `match_request.matched` + `booking.created` inside one tx; expire/cancel/sweep wrapped in `withTx`), `routes/bookings.js` (transition helper wrapped in `withTx`, emits `booking.{accepted,in_progress,completed,cancelled}`), `routes/match.js` (POST `/api/match` wrapped, emits `match_request.created`), `routes/payments.js` (Peach webhook wrapped, emits `payment.{succeeded,failed}` based on result code).

## Where to start the next session

**My pick: audit log primitive + `audit_log_query` MCP tool.**

It's Memo 2's highest-leverage want and the natural Phase 2 of the agent-native bet — and it plugs into the same `emitEvent` abstraction as a second downstream consumer (no re-architecture needed; just a new side effect inside `emitEvent`). Two passes:

1. **`audit_log` table + emitEvent integration** — every event emit writes an audit row alongside the webhook deliveries. New columns: `agent_id` (nullable, from API key), `actor_user_id` (the req.user.id at emit time), `action`, `resource_type`, `resource_id`, `payload`, `created_at`. Same tx as the resource mutation.
2. **`audit_log_query(actor_user_id?, agent_id?, action?, resource_type?, since?)` admin MCP tool** — paginated, scope-gated `admin:full`. Memo 2: "the marketplace won't trust me if it can't audit me."

Estimated effort: 3–4 hours (smaller than webhooks because the abstraction's already there).

- Pin-it map UX (Memo 1: "I couldn't find your house" cancellations) — ~1 evening, pure mobile work.
- Offline persistence on mobile (load-shedding survival) — ~1 evening, AsyncStorage cache for active bookings.
